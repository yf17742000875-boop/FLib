# 每周高质量机器学习研报

时间范围：2026-05-15 至 2026-05-22

## 1. 本周核心结论

本周最清晰的信号不是“模型又涨分了”，而是机器学习系统正在进一步工程化为“可执行、可追踪、可约束”的工作流。

1. Agent 方向从“会调用工具”继续转向“把技能写成可执行控制逻辑”，HASP 这类工作说明，下一阶段的竞争点不只是推理能力，而是技能如何在 agent loop 里真正生效。
2. RAG 正在从“LLM 负责一切索引加工”转向“结构先行、LLM 只做必要补充”，ContextRAG 体现的是成本和可扩展性优先的索引范式。
3. 优化器与训练技巧开始更强调参数矩阵层面的差异化处理，AMO 说明统一调度不再是默认最优，几何感知和按层/按矩阵分配预算会成为更常见思路。
4. 生成式 AI 的部署栈越来越重视 provenance、远程协作、长任务接续和验证机制，说明“把模型放进生产”已经等价于“把观测、控制和可信链路一起放进去”。

## 2. 重点工具 / 方法 / 工程技巧

### 2.1 Empirical Research Assistance（ERA）

Google Research 发布的 ERA 是一个面向专家级科学编码的 AI 工具，用 Gemini 辅助编写和优化科学代码，目标是缩短“写实验代码 - 跑实验 - 改实验”这个最耗时的循环。对做科研型 ML 的人来说，它的价值不在于替代研究判断，而在于把重复的实验脚手架、代码改写和调参工作压缩掉，让研究者把注意力保留给假设、变量控制和结果解释。  
来源： [Google Research blog](https://research.google/blog/empirical-research-assistance-era-from-nature-publication-to-catalyzing-computational-discovery/)

### 2.2 Codex 的移动端远程接续与 Goal mode

OpenAI 本周把 Codex 的移动端远程接入、Goal mode、Appshots 和锁屏持续运行都推进到了更实用的状态。工程价值很直接：长任务 agent 不再要求你“坐在机器前等它跑完”，而是可以在手机上接着审阅、批准、改方向，适合做长时间调研、代码修复和多阶段实验排查。  
来源： [ChatGPT release notes](https://help.openai.com/en/articles/6825453-chatgpt-release-notes)

### 2.3 内容 provenance 栈：C2PA + SynthID + 公共验证

OpenAI 在 5 月 19 日更新了内容 provenance 策略，把 C2PA 合规、Google SynthID 水印和公共验证工具一起纳入生成式内容链路。工程上这说明：模型能力只是第一层，真正可部署的生成系统还要补上“可识别、可验证、可跨平台保留信号”的基础设施，否则生成内容越强，治理与追溯成本越高。  
来源： [OpenAI provenance announcement](https://openai.com/index/advancing-content-provenance/)

## 3. 本周高质量论文精读

### 3.1 Harnessing LLM Agents with Skill Programs

HASP 把“技能”从文本建议升级为可执行的 Program Functions（PFs），核心变化是让技能不再只是提示词级别的软约束，而是在失败状态下真正插入 agent loop，修改下一步动作或补充纠偏上下文。它的亮点是模块化很强：既能做推理时干预，也能做后训练监督，还能通过 teacher-reviewed PFs 做自我进化。实验上，它在 web-search reasoning 上相对 ReAct Agent 有 25% 的提升，相对 Search-R1 也有 30.4% 的提升，说明“执行型技能”比“口头型技能”更可能成为 agent 的实用形态。局限也很明确：这类方法依赖稳定的技能库演化与触发条件设计，落地时维护成本不会低。  
来源： [arXiv:2605.17734](https://arxiv.org/abs/2605.17734)

### 3.2 ContextRAG: Extraction-Free Hierarchical Graph Construction for Retrieval-Augmented Generation

ContextRAG 试图解决图 RAG 中最贵的那部分：索引阶段的实体、关系和摘要抽取。方法上，它不让 LLM 直接写图边，而是用 chunk embedding、残差量化 k-means、形式概念分析和 Lukasiewicz residuated logic 构造模糊概念图，再由软的 join / meet 诱导桥接节点。这个思路非常工程化：在 130 任务的 UltraDomain 子集上只用了 30 次 LLM 调用和 22,073 tokens，而对照的 HiRAG 复现实验在更小子集上就消耗了 870 次调用和 3.54M tokens 并最终失败。结果上它拿到 33.6% overall F1、multi-hop 36.8% F1，说明“少用 LLM 做索引加工”不是单纯省钱，也可能更稳。局限在于它的收益高度依赖图结构质量，且当前分析更多是相关性，不是因果证明。  
来源： [arXiv:2605.19735](https://arxiv.org/abs/2605.19735)

### 3.3 AMO: Adaptive Muon Orthogonalization

AMO 关注的是 Muon 优化器里一个很具体但很实在的问题：不同参数矩阵的正交化难度并不一样，统一的 Newton-Schulz 调度会让部分层“吃不饱”或“过度算”。作者用早期训练阶段的几何统计来观察各类矩阵差异，然后在后续训练里按 operator type 分配 NS 预算，相当于把“优化器超参数”从全局常数改成了几何感知的资源调度。实验收益不算夸张，但很一致：在 Llama3.1-1.4B 和 Qwen3-1.7B 上都超过了统一调度的 Muon，并在 12 项下游任务的平均表现上取得提升。这个工作的重要性在于它强化了一个趋势：训练技巧不再只是调学习率，而是在更细粒度上根据参数几何和训练阶段动态分配计算预算。  
来源： [arXiv:2605.17806](https://arxiv.org/abs/2605.17806)

## 4. 机器学习发展脉络与趋势判断

本周的趋势可以概括为四条主线。

第一，agent 正从“推理型模型”走向“执行型系统”。真正有价值的不是模型是否会思考，而是它能否把经验封装成可执行的控制策略、在失败状态触发正确干预，并且把长任务变成可持续的工作流。

第二，RAG、数据加工和索引构建正在经历一次“结构优先”的重写。过去常见的范式是让 LLM 参与尽可能多的中间步骤，现在更像是在问：哪些步骤可以由更便宜、更稳定的结构化算法接管，哪些步骤才值得调用 LLM。

第三，优化与训练在向“几何感知”和“资源分配”演化。统一的学习率、统一的正交化调度、统一的策略奖励，越来越经不起复杂模型和复杂训练阶段的压力，按矩阵、按层、按阶段分配预算会变得更重要。

第四，生成式系统的边界正在外扩到 provenance、验证、远程协作和持续接续执行。也就是说，模型能力本身已经不是唯一瓶颈，真正的系统竞争在于：谁能把生成、审阅、溯源、控制和协作整合成一个可信链路。

## 5. 哪些方向值得下周继续追踪

1. Skill program / executable skill library 是否会成为 agent 的主流中间层，还是只停留在少数高价值任务上。
2. 是否会出现更多“结构先行”的 RAG 索引方法，继续压低 LLM 在索引阶段的 token 和 wall-clock 成本。
3. Muon、orthogonalization、second-order-ish 训练技巧是否会继续出现矩阵级、层级自适应版本。
4. provenance、watermark、verification 这类生成式内容治理能力是否会进一步变成标准能力，而不是安全附加项。
5. 长任务 agent 的产品化能力，尤其是移动端接续、远程锁屏运行、目标驱动模式，会不会明显改变真实工作流。
