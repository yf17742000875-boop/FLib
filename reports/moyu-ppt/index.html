Context 02 / 08
A better framing
“摸鱼”常常不是对工作的反抗，而是对失控节奏的一次微小自救。
Observation / Office Ecology

为什么这个题目值得讲

在很多团队里，效率讨论只看产出，不看注意力的恢复机制。于是大家表面在线，实际分散；表面很忙，实则在碎片化地流失精力。

  • 摸鱼是注意力负债的症状，不是根因。
  • 高压和空转同时存在时，最容易出现“假忙碌”。
  • 组织真正需要优化的，是节奏设计，而不是道德审判。

Scene Types 04 / 08
Four species

办公室摸鱼，不是一种，是四种

Type 01

回血型

做完一段高负荷任务后短暂发呆、刷窗外、接杯水，让认知栈清空。

Type 02

逃逸型

任务模糊、反馈稀薄，于是人自动漂向最轻松、最即时满足的页面。

Type 03

表演型

看起来一直在线，实则用存在感代替进展，用消息响应代替真正推进。

Type 04

学习型

去看一点和手头工作若即若离的东西，表面偏航，实际上在补长期带宽。

不要一概而论 先分型再判断
Before / After 05 / 08
旧叙事

摸鱼 = 人不行

  • 默认问题出在个人态度。
  • 通过更严的在线监管解决焦虑。
  • 让所有空白时间都显得可疑。
  • 结果是更重的表演，更轻的创造。
新叙事

摸鱼 = 系统在求助

  • 先看任务定义、会议密度和反馈周期。
  • 把休息显性化，而不是把偷闲地下化。
  • 允许短暂离线，换取更长时间的深度推进。
  • 结果是更少假忙碌，更高质量的专注。
Moral lens to system lens

Workflow 07 / 08
Practical protocol

把摸鱼升级成“低摩擦恢复机制”

01

设窗口

允许自己每完成一段深度任务后离开 10-15 分钟，而不是随机碎掉一整天。

02

降噪音

把最容易制造假在线的通知、群聊和悬浮提示关掉一层。

03

记产出

用可交付结果衡量今天，而不是用“坐了多久、亮了多久”衡量。

04

留余地

真正可持续的团队，日程里总会有一点空白，不把每分钟都榨干。

Not slacking but recovery by design
Takeaway 08 / 08
Final line

最成熟的团队，不会消灭摸鱼。
它会让人不必偷偷摸鱼。

当恢复被制度化，专注才有可能被长期维持。与其把所有停顿都视为失败，不如把节奏设计成一种生产力基础设施。
